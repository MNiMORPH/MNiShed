# notes/

Tracked development notes: session **handoffs** (context carried between working
sessions) and analysis **reports**. Kept in version history so old directives and
findings stay recoverable, but out of the way — these are *not* part of the
published user documentation (that lives in `docs/source/`).

Nothing here is imported by the package or required to run it.

- `HANDOFF_*.md` — session-bridging notes on a specific thread (what was found,
  what's left, the reasoning to resume from).
- `REPORT_*.md` — standalone write-ups of a calibration or analysis run.
